## 二、推荐的 AI 工作流

1. 「渐进式提示」工作流

初始想法 → 细化需求 → 迭代优化 → 最终输出

- 从模糊想法开始

- 让AI帮助细化需求

- 多轮对话迭代

2. 「AI先行，人工校验」工作流

AI生成初稿 → 人工审查 → AI修改 → 最终确认

- 让AI完成80%的基础工作

- 人工专注20%的关键决策

- 适用于文档、代码、邮件等

3. 「分治协作」工作流

任务分解 → 多个AI并行处理 → 人工整合 → 质量检查

- 复杂任务拆解为子任务

- 用不同AI工具处理各自擅长部分

- 人工统筹整合结果

## 三、AI 工作方法

1. 提示工程核心原则

- 具体化：明确输入、输出、约束条件

- 上下文先行：先给背景，再提要求

- 迭代优化：不要指望一次完美输出

2. 编写有效提示的框架

角色设定 + 任务描述 + 输入数据 + 输出格式 + 约束条件 + 示例

示例：

你是一位资深Python工程师（角色）

请优化以下函数的性能（任务）

输入代码：[代码]（输入数据）

要求输出：优化后的代码 + 时间复杂度分析（输出格式）

不要改变函数接口（约束条件）

4. 避免 AI 常见陷阱

- ❌ 过度依赖AI，失去独立思考能力

- ❌ 不验证AI输出的准确性

- ❌ 提示过于笼统

- ❌ 忽略AI的安全和隐私限制

- ✅ AI作为副驾驶，人类保持主导

- ✅ 批判性思维审查AI输出

- ✅ 保护敏感数据

## 四、实用工作流模板

学习新技术

1. 让AI生成学习大纲

2. 让AI解释核心概念

3. 让AI提供实践练习

4. 用AI解答疑问

5. 让AI测试你的理解

编写技术文档

1. 让AI生成文档结构

2. 逐章填充内容（AI辅助）

3. 让AI检查一致性和完整性

4. 让AI优化表达清晰度

解决编程问题

1. 向AI清晰描述问题

2. 让AI提供多种解决方案

3. 讨论各方案优劣

4. 选择方案并让AI生成代码

5. 让AI进行代码审查
